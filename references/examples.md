# 中大鸭鸭 · 示例对话

与 [`SKILL.md`](SKILL.md) 主说明配套，演示记忆、搜索与改名等流程。

## 示例1：记忆命中

```
用户：松涛园早餐有什么
→ 调用 duck.py recall "松涛园早餐"
→ 命中：{"hit": true, "canonical": "松涛园在东区，二楼叉烧出名"}
→ 直接回答："东区的松涛园，二楼叉烧超棒！"
```

## 示例2：模糊匹配命中

```
用户：图书馆借书怎么借
→ 调用 duck.py recall "图书馆借书"
→ 种子记忆命中：{"hit": true, "source": "seed", "canonical": "持校园卡在一楼柜台借阅，一般可借30天"}
→ 直接回答："持校园卡在一楼柜台借阅，一般可借30天～"
```

## 示例3：搜索后记住

```
用户：图书馆怎么借书
→ duck.py recall → hit=false（未命中）
→ 搜索"中山大学 南校 图书馆借书"
→ 回答："持校园卡在一楼柜台借阅，一般可借30天～"
→ 判断：这条知识普遍适用，值得记住
→ 调用 duck.py remember "图书馆借书" "持校园卡在一楼柜台借阅，一般可借30天" "图书馆 借书 校园卡 期限" "南校"
→ 回答时自然说："这个我记住了，下次直接告诉你～"
```

## 示例4：搜索后不记住

```
用户：今天松涛园有什么菜
→ 搜索"中山大学 南校 松涛园 今天菜单"
→ 回答："今天二三楼好像有糖醋排骨～"
→ 判断：时效性太强，不值得记住
→ 不调用 remember
```

## 示例5：遗忘

```
用户：忘了图书馆借书的事
→ 调用 duck.py forget "图书馆借书"
→ 回复："🧹 好，「图书馆借书」我忘了～"
```

## 示例6：改名校验

```
用户：/改名 超级长的昵称超过八个字
→ duck.py rename → 失败："🦆 昵称太长了，8个字以内哦"

用户：/改名 阿花*123
→ duck.py rename → 失败："🦆 昵称只能包含中文、英文和数字"

用户：/改名 阿花Pro
→ duck.py rename → 成功："🦆 改名成功！以后就叫「阿花Pro」啦～"
```
